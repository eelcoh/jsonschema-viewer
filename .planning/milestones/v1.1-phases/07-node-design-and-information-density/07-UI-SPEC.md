# Phase 7 — UI Design Contract

> Visual and interaction contract for Node Design and Information Density.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase operates entirely in SVG space — no HTML components, no shadcn, no Tailwind.
> All design tokens are expressed as SVG attributes and Elm constants in `Render.Theme`.

## Typography

All SVG text uses `fontFamily "Monospace"` at `fontSize "12"`. This is fixed by the existing coordinate-threading system — changing font metrics would break all node width calculations.

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Node icon text | 12px | 700 (bold) | n/a (SVG `dominantBaseline "middle"`) | `iconGeneric`, `iconGraph IInt` |
| Node name text — required | 12px | 700 | n/a | `viewNameGraph` with weight "700" |
| Node name text — optional | 12px | 400 | n/a | `viewNameGraph` with weight "400" |
| Overlay label (row key) | 11px | 400 | 1.4 | Hover overlay SVG text |
| Overlay value | 11px | 700 | 1.4 | Hover overlay SVG text |
| HTML app title | 20px | 600 | 1.2 | `.app-title` CSS (existing, do not change) |
| HTML body / controls | 14px | 400 | 1.5 | `.example-btn`, `.error-body` CSS (existing) |

**Overlay font choice:** Monospace 11px. Keeps visual consistency with nodes. Smaller than node text (12px) to signal secondary/metadata role. 11px is legible at typical screen resolutions and fits within the overlay panel width.

Source: codebase `src/Render/Svg.elm` lines 732–736, 847–853; `src/main.css`

## Color

All colors live in `src/Render/Theme.elm`. New constants required by this phase are listed with the name they MUST use in that module.

| Role | Value | Usage | Theme constant |
|------|-------|-------|----------------|
| Dominant (60%) | `#1a2332` | SVG background, dark navy | `background` (existing) |
| Secondary (30%) | `#2a3a52` | Grid dots, overlay panel background | `gridDot` (existing) / `overlayBg` (new) |
| Node border — default (optional) | `#a0c4e8` | Optional property node outlines, separator lines | `nodeBorder` (existing) |
| Node border — required | `#e8a020` | Required property node outlines only | `requiredBorder` (new) — see amber spec below |
| Node text | `#e8f0f8` | All node label text, icon glyphs | `nodeText` (existing) |
| Connector lines | `#4a6a8a` | Bezier connector paths | `connector` (existing) |
| Overlay background | `#0f1e30` | Hover overlay panel fill | `overlayBg` (new) |
| Overlay border | `#3a5a7a` | Hover overlay panel stroke | `overlayBorder` (new) |
| Overlay key text | `#8ab0d0` | Row label text in overlay | `overlayKeyText` (new) |
| Overlay value text | `#e8f0f8` | Row value text in overlay | `nodeText` (reuse existing) |
| Destructive (errors only) | `#ff8591` | Error heading (existing, no change) | `errorHeading` (existing) |

**Accent reserved for:** Required property node borders only (`requiredBorder`). No other element uses amber `#e8a020`.

**Amber spec rationale:** `#e8a020` sits at ~50° hue (amber-orange) against the dark navy background (`#1a2332`). Its contrast ratio against transparent fill is approximately 6.5:1 — above WCAG AA 4.5:1 for text-equivalent elements. It clearly differs from the default blue-grey border `#a0c4e8` (distinct hue, slightly higher luminance). Warm amber reads as "attention/required" on a cool-toned blueprint palette.

**Overlay background spec:** `#0f1e30` is slightly darker than `gridDot` (`#2a3a52`) and the main background (`#1a2332`), creating a subtle layered depth. Contrast for `overlayKeyText` (#8ab0d0) against `#0f1e30` is approximately 5.1:1 (AA pass).

Source: CONTEXT.md D-01, D-03; Claude's Discretion (color values); `src/Render/Theme.elm`

## Icon Design Contract

Icons are monospace text glyphs rendered at 12px in SVG. Each icon replaces the 'S' glyph in the type-icon position of the pill (leftmost cell, before the separator line).

| Icon variant | Glyph text | Applies when |
|---|---|---|
| `IStr` | `S` | String with no format and no enum (existing, unchanged) |
| `IEmail` | `@` | `format = Just Email` |
| `IDateTime` | `dt` | `format = Just DateTime` |
| `IHostname` | `dns` | `format = Just Hostname` |
| `IIpv4` | `ip4` | `format = Just Ipv4` |
| `IIpv6` | `ip6` | `format = Just Ipv6` |
| `IUri` | `url` | `format = Just Uri` |
| `ICustom s` | first 3 chars of `s`, lowercased | `format = Just (Custom s)` — truncate to 3 chars to fit pill icon cell |
| `IEnum` | `Enum` | `enum = Just (...)` on ANY base type (overrides type icon) |
| `IInt` | `I` | Integer with no enum (existing, unchanged) |
| `IFloat` | `F` | Number with no enum (existing, unchanged) |
| `IBool` | `B` | Boolean with no enum (existing, unchanged) |
| `INull` | `Null` | Null type (existing, unchanged) |
| `IObject` | `{..}` | Object type (existing, unchanged) |
| `IList` | `[..]` | Array type (existing, unchanged) |
| `IRef s` | `s` | $ref node (existing, unchanged) |

**Icon width budget:** The icon cell width is `computeTextWidth glyph`. Single-char icons (`@`, `S`, `I`, `B`, `F`) = 7.2px. Two-char icons (`dt`, `ip4`, `ip6`, `url`, `dns`, `Enum`) = 14.4–36px. `Enum` = 28.8px. Pill auto-sizes via existing `iconRect` layout — no fixed widths needed.

**Priority rule:** `IEnum` takes precedence over format icons. If a node has both `enum` and `format`, render `IEnum` and show format in the hover overlay.

Source: CONTEXT.md D-04, D-05, D-06, D-07; Claude's Discretion (specific glyphs)

## Hover Overlay Design Contract

### Trigger

- `Svg.Events.onMouseEnter` fires `HoverNode nodePath` on the pill group `<g>` element.
- `Svg.Events.onMouseLeave` fires `UnhoverNode` on the same group.
- Only one overlay visible at a time (`hoveredNode : Maybe String` in Model).
- Overlay appears only when the node has at least one of: `description`, constraints, `enum` values, or format detail (for `IEnum` nodes showing base type).

### Positioning

- Overlay top-left corner anchors at `(nodeRightX + 8, nodeTopY)`.
- If the overlay would extend beyond the diagram's right edge (estimated), flip to render left of the node: `(nodeLeftX - overlayWidth - 8, nodeTopY)`.
- The overlay is rendered LAST in the SVG tree (after all nodes), ensuring it paints on top regardless of SVG z-order.
- Overlay does NOT participate in coordinate-threading layout — it is absolutely positioned in SVG space and has zero effect on sibling node positions.

### Overlay Panel Dimensions

| Property | Value |
|----------|-------|
| Minimum width | 160px |
| Maximum width | 300px |
| Horizontal padding | 12px each side |
| Vertical padding | 8px top, 8px bottom |
| Row height | 18px per metadata row |
| Corner radius | 4px (`rx="4" ry="4"`) |
| Border stroke width | 1px |
| Background | `Theme.overlayBg` (`#0f1e30`) |
| Border color | `Theme.overlayBorder` (`#3a5a7a`) |

### Overlay Content Layout

Each metadata row is a key–value pair rendered as two SVG `<text>` elements on the same horizontal baseline:

```
[key_label]   [value_text]
```

Key label: left-aligned, `Theme.overlayKeyText` (`#8ab0d0`), 11px, weight 400.
Value text: follows key label with 8px gap, `Theme.nodeText` (`#e8f0f8`), 11px, weight 700.

Row order (top to bottom, skip if absent):

1. `type` — base type name (shown when icon is `IEnum` or `ICustom`, to surface hidden base type)
2. `format` — format string (shown when icon is `IEnum` to reveal the overridden format)
3. `desc` — full description text (wrap at overlay max-width - 24px, no truncation per D-12)
4. `enum` — comma-separated values, max 5 shown then `+N more` (e.g., `"a", "b", "c", +2 more`)
5. `min` / `max` — numeric minimum / maximum (integer and number types)
6. `minLen` / `maxLen` — minLength / maxLength (string type)
7. `pattern` — regex pattern string (truncated at 40 chars with `…` if longer)

Long description text wraps onto multiple lines. Each wrap line adds another 18px to the overlay height. The overlay height is computed before rendering.

### Text Wrapping for Descriptions

Description text wraps at `overlayMaxWidth - 24px` (i.e., 276px for 300px overlay). Use a fixed-width estimate: at 11px monospace, approximately 6.5px per character. A 276px column fits ~42 characters per line.

### No Overlay Conditions

Do NOT render an overlay for nodes that have none of the above metadata fields populated. `onMouseEnter` still fires `HoverNode` but `viewHoverOverlay` returns `Svg.g [] []` when no metadata exists.

Source: CONTEXT.md D-09, D-10, D-11, D-12, D-13; Claude's Discretion (dimensions, positioning, row layout)

## Required vs Optional Border Contract

| Property state | Border color | Font weight | Source |
|----------------|-------------|-------------|--------|
| Required | `Theme.requiredBorder` = `#e8a020` (amber) | 700 (bold) | CONTEXT.md D-01, D-02 |
| Optional | `Theme.nodeBorder` = `#a0c4e8` (blue-grey) | 400 (normal) | CONTEXT.md D-01, D-02 |

The `iconRect` function gains an `isRequired : Bool` parameter. When `True`, it substitutes `Theme.requiredBorder` for `Theme.nodeBorder` in the `border` attribute of the outer `<rect>`.

The separator line (`separatorGraph`) uses `Theme.nodeBorder` regardless of required/optional status — it is a visual divider, not a status indicator.

Source: CONTEXT.md D-01, D-02, D-03

## Copywriting Contract

This phase adds no new HTML copy. All new text is metadata surfaced directly from the schema document. The following contracts apply to overlay-rendered text:

| Element | Copy |
|---------|------|
| Overlay row — no description | Row omitted entirely (no "No description" placeholder) |
| Overlay row — no constraints | Rows omitted entirely |
| Enum overflow | `+{N} more` (e.g., `+3 more`) — N is the count of values beyond the 5 shown |
| Pattern truncation suffix | `…` (ellipsis character U+2026) |
| Empty diagram (no schema loaded) | Existing copy — not changed by this phase |
| Parse error | Existing error display — not changed by this phase |

No destructive actions in this phase. No confirmations required.

Source: CONTEXT.md D-08, D-12, D-13; Claude's Discretion (overflow copy)

## State Contract

### New Model Fields

```elm
-- In Main.elm Model record:
hoveredNode : Maybe String
```

`hoveredNode` holds the `path : String` of the currently hovered node, or `Nothing` if no node is hovered.

### New Messages

```elm
-- In Main.elm Msg:
HoverNode String   -- node path being entered
UnhoverNode        -- mouse left node
```

### View Signature Change

`Render.Svg.view` receives the hover state and a hover message constructor:

```elm
view :
    (String -> msg)      -- toggleMsg (existing)
    -> (String -> msg)   -- hoverMsg (new)
    -> msg               -- unhoverMsg (new)
    -> Maybe String      -- hoveredNode (new)
    -> Set String        -- collapsedNodes (existing)
    -> Definitions       -- defs (existing)
    -> Schema            -- schema (existing)
    -> Html.Html msg
```

The overlay element is rendered by a dedicated function called at the end of the top-level `view`, after `schemaView`:

```elm
, schemaView
, overlayView  -- rendered last → paints on top
```
