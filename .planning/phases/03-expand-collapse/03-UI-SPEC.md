## Interaction Contract

This section is specific to Phase 3 — captures click behavior, cursor, and state change. (Standard UI-SPEC template does not have this section; added because INTR-01 is purely interaction.)

### Click Targets

| Node type | Clickable | Cursor on hover | Toggle behavior |
|-----------|-----------|-----------------|-----------------|
| Object | Yes | `pointer` | Hides/shows all child property nodes |
| Array | Yes | `pointer` | Hides/shows items schema |
| OneOf / AnyOf / AllOf | Yes | `pointer` | Hides/shows all sub-schema nodes |
| Ref (not expanded) | Yes | `pointer` | Expands definition inline; subsequent click collapses |
| Ref (cycle guard hit) | No | `default` | Cycle pill (↺) is not interactive |
| String | No | `default` | Leaf — no interaction |
| Integer | No | `default` | Leaf — no interaction |
| Number | No | `default` | Leaf — no interaction |
| Boolean | No | `default` | Leaf — no interaction |
| Null | No | `default` | Leaf — no interaction |

### Visual State: Collapsed Node

A collapsed container node renders its own pill at full opacity with no visual change to the pill itself. Children are simply not rendered — no animation, no transition, no collapsed indicator glyph added to the pill. The absence of children is the only visual signal.

Rationale from D-08/D-09 (CONTEXT.md): the pill is the click target; no +/- icon; no hover color change. This is consistent with the current `roundRect`/`iconRect` pill design which has no hover state in existing code.

### Visual State: Expanded $ref Node

An expanded $ref node renders its referenced definition's schema inline, using the same layout and pill styles as any inline schema. The collapsed $ref pill (`*` icon + definition name) is replaced by the full inline subtree. No wrapper, tint, or border distinguishes the expanded $ref content (D-06).

### Pointer Cursor

Applied via `SvgA.cursor "pointer"` on the wrapping `<g>` element for each container pill. Leaf nodes do not receive this attribute. Existing pill text elements already carry `cursor: pointer` — this is extended to the outer `<g>` so the entire pill bounding box is covered.

### No Animation

No CSS transition or SVG animation on collapse/expand. Layout reflows immediately on the next Elm render cycle. (Deferred to v2 or Phase 4 at Claude's discretion — not required for INTR-01.)

## Copywriting Contract

This phase adds no new user-facing copy beyond what already exists. The expand/collapse interaction is purely visual (clicking SVG nodes). There are no labels, tooltips, or confirmation dialogs in this phase.

| Element | Copy |
|---------|------|
| Primary CTA | None — interaction is click-to-toggle on SVG nodes; no button label |
| Empty state heading | "Paste a JSON Schema to see the diagram" (existing, unchanged) |
| Empty state body | "Use the textarea on the left to enter a schema, or pick an example above." (existing, unchanged) |
| Error state | "Invalid schema — {parse error message}" (existing, unchanged) |
| Destructive confirmation | None — no destructive actions in this phase |

Note: Collapse state resets silently on schema re-parse (D-03). No toast, no warning message — reset is implicit and expected.
