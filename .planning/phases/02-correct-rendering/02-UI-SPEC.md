## Spacing Scale

This phase makes no changes to the HTML/CSS layout. All spacing changes are in SVG coordinate space. The existing CSS spacing tokens are retained as-is and listed here for completeness.

### CSS Layout Spacing (existing — read-only for this phase)

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing — toolbar gap, example button padding |
| md | 16px | Default element spacing — toolbar horizontal padding, textarea padding |
| lg | 24px | Not explicitly used in current CSS |
| xl | 32px | Error container padding |
| 2xl | 48px | Toolbar height (48px) |
| 3xl | 64px | Not used |

### SVG Coordinate Spacing (modified in this phase)

| Name | Value | Usage |
|------|-------|-------|
| `ySpace` | 10px | Vertical gap between sibling nodes (existing constant in Render/Svg.elm) |
| `pillHeight` | 28px | Height of all pill-shaped nodes (existing constant) |
| `siblingGap` | 10px | Horizontal gap between parent node and its children (existing, used as `w + 10`) |
| `viewBox padding` | 20px | Padding added around the computed diagram extent in the dynamic viewBox (Claude's discretion, per CONTEXT.md) |

**Source:** `src/Render/Svg.elm` constants `ySpace = 10`, `pillHeight = 28`; RESEARCH.md Pattern 1 (padding = 20).

**Exceptions — SVG coordinate values not on the 8-point CSS scale:**

`ySpace = 10px` and `siblingGap = 10px` are pre-existing SVG coordinate constants inherited from the original renderer. They are not CSS layout values and are not part of the HTML grid system. The 10px value was chosen for visual density in diagram layout — it produces compact, readable node spacing in the SVG coordinate plane where the unit is a logical pixel of the viewBox, not a screen pixel. These values are retained unchanged in this phase and are intentionally exempt from the 8-point CSS spacing scale.

`viewBox padding = 20px` similarly lives in SVG coordinate space. It provides a comfortable margin around the diagram extent. It is not a CSS layout value.

Exceptions: The SVG `width` and `height` attributes change from hardcoded `"520"` to `"100%"` — this is not a spacing value but a viewport sizing change per D-09.

## Typography

Two rendering contexts exist: CSS (HTML shell) and SVG (diagram nodes). They are independent.

### CSS Typography (existing — read-only for this phase)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / UI | 14px | 400 (regular) | 1.0 (single-line controls) |
| Textarea content | 13px | 400 | 1.6 |
| Heading / App title | 20px | 600 (semibold) | 1.2 |
| Error heading | 20px | 600 | inherited |
| Error body | 14px | 400 | inherited |
| Error detail (mono) | 13px | 400 | inherited |

**Source:** `src/main.css` — `.app-title`, `.example-btn`, `.schema-textarea`, `.error-heading`, `.error-body`, `.error-detail`.

### SVG Node Typography (modified in this phase — REND-03)

| Role | Size | Weight | Font | Usage |
|------|------|--------|------|-------|
| Required property name | 12px | 700 (bold) | Monospace | Property names that are required in their parent object |
| Optional property name | 12px | 400 (regular) | Monospace | Property names that are optional in their parent object |
| Type icon label | 12px | 400 | Monospace | Icon section of iconRect nodes (`{..}`, `[..]`, `S`, `I`, `F`, `B`) |
| $ref node label | 12px | 400 | Monospace | Definition name shown in the name section of $ref iconRect nodes |
| Cycle indicator suffix | 12px | 400 | Monospace | ` ↺` appended to $ref label when circular reference detected |

**Decision source:** D-04 (bold for required, normal for optional), D-05 (no color/icon difference — weight only), D-07 (↺ suffix for cycles). RESEARCH.md Pattern 3.

**Implementation note:** The weight distinction is applied only to the immediate property name label in `viewProperty`. It does not propagate to nested schemas. See RESEARCH.md Pitfall 5.

## SVG Viewport Contract

This is the central visual change of Phase 2 (REND-02).

| Property | Before | After | Decision |
|----------|--------|-------|----------|
| `SvgA.width` | `"520"` | `"100%"` | D-09 |
| `SvgA.height` | `"520"` | `"100%"` | D-09 |
| `SvgA.viewBox` | `"0 0 520 520"` | `"0 0 {w+20} {h+20}"` where `w, h` are the `Dimensions` returned by the top-level `viewSchema` call | D-08, D-10 |
| Container | `.diagram-panel` with `overflow: auto` | Unchanged — `overflow: auto` allows scrolling when diagram exceeds viewport | Phase 1 CSS |

**viewBox string format:** `"0 0 " ++ String.fromFloat (w + 20) ++ " " ++ String.fromFloat (h + 20)`. Four space-separated floats. Starting position is always `0 0` because `viewSchema` is called at `( 0, 0 )`. See RESEARCH.md Pitfall 2 and Pitfall 3.
