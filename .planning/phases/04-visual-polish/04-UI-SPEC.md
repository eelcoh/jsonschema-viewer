# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Visual Polish. Generated by gsd-ui-researcher.
> This phase adds SVG connector lines and a dashed border style for $ref nodes entirely inside Elm SVG code. There is no HTML component library, no CSS framework, and no shadcn. All design tokens are expressed as Elm constants and SVG attributes.

## Spacing Scale

The SVG diagram uses a coordinate-threading layout model, not a CSS grid. Spacing is defined as numeric pixel values threaded through view functions. The following values are extracted from existing code and serve as the canonical spacing contract for Phase 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon inner padding |
| sm | 8px | Toolbar button horizontal padding; gap between elements |
| md | 16px | Default horizontal padding (toolbar, textarea, error container) |
| lg | 24px | Section-level spacing in error views |
| xl | 32px | Error container padding; collapse toggle button size |
| ySpace | 10px | Vertical gap between sibling nodes in the SVG diagram (existing constant) |
| pillHeight | 28px | Height of every node pill in the SVG diagram (existing constant) |
| pillInnerPad | 10px | Horizontal padding inside the pill icon area (existing `space` constant) |

Exceptions:
- Toolbar height is 48px (matches existing `.toolbar` CSS, not on the 8-point scale — existing, do not change).
- `ySpace` (10px) is existing and must not change — layout arithmetic in `viewProperties`/`viewItems` depends on it.
- `pillHeight` (28px) is existing and must not change.

Source: `src/main.css`, `src/Render/Svg.elm` (lines 28–31, 512–519).

### Phase 4 New Color Decisions

| Element | Value | Rationale |
|---------|-------|-----------|
| Connector line stroke | `#8baed6` | Mid-tone between `darkClr` (`#3972ce`) and `lightClr` (`#e6e6e6`); visible on the white diagram background without competing with node borders. Claude's discretion — D-03. |
| Connector line stroke-width | `1.5` | Thin enough to read as structural guide, thick enough to be visible at scale. |
| Connector stroke-linecap | `round` | Matches `separatorGraph` existing pattern (`strokeLinecap "Round"`). |
| $ref dashed border pattern | `strokeDasharray "5 3"` | 5px dash, 3px gap — readable at 12px node scale; distinct from solid borders without visual noise. Claude's discretion. |
| Cycle pill dashed border | `strokeDasharray "5 3"` | Same pattern as $ref per D-06. |

Accent reserved for: active example button fill, textarea focus ring, collapse toggle icon color. Not used for connector lines or dashed borders (those use the mid-tone `#8baed6` and the existing `lightClr` respectively).

## SVG Visual Contracts

These are the specific SVG attributes that Phase 4 introduces or modifies. The executor MUST treat these as binding.

### Connector Lines (VIS-01)

```
Element:        SVG <path> with cubic bezier (C command)
Stroke:         #8baed6
Stroke-width:   1.5
Stroke-linecap: round
Fill:           none
Start point:    right-center of parent pill  → (parentX + parentWidth, parentY + pillHeight/2)
End point:      left-center of child pill    → (childX, childY + pillHeight/2)
Control points: CP1 = (startX + horizontalOffset, startY)
                CP2 = (endX - horizontalOffset, endY)
                where horizontalOffset = (endX - startX) * 0.5
Visibility:     only rendered when node is expanded (children are visible)
Path d attr:    "M {startX} {startY} C {cp1x} {cp1y} {cp2x} {cp2y} {endX} {endY}"
```

The connector line for each child is generated inside `viewProperties` / `viewItems` alongside each child render call, using the parent's right-edge coordinate and each child's returned left-edge coordinate.

Source: CONTEXT.md D-01, D-04; Claude's discretion for color/thickness (D-03).

### $ref Dashed Border (VIS-02)

```
Modified element:   the <rect> inside iconRect when icon is IRef _
Change:             add SvgA.strokeDasharray "5 3"
No other changes:   fill (darkClr), stroke color (lightClr), strokeWidth (0.2), rx/ry (2) unchanged
Also applies to:    cycle pill (the rect rendered for circular $ref nodes) — same dasharray
```

Source: CONTEXT.md D-05, D-06.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Expand node | Connector lines to children appear immediately with the child nodes |
| Collapse node | Connector lines to children disappear immediately with the child nodes |
| Click $ref node (expanded) | Inline children appear with connector lines (existing Phase 3 behavior extended) |
| Click $ref node (cycle) | Cycle pill is not clickable (existing Phase 3 behavior unchanged) |
| Hover connector line | No hover effect — purely structural, not interactive |
