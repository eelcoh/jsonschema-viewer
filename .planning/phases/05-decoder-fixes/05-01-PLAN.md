**1. Add `normalizeRef` helper** (new top-level function):
```elm
normalizeRef : String -> String
normalizeRef ref =
    if String.startsWith "#/$defs/" ref then
        "#/definitions/" ++ String.dropLeft 8 ref
    else
        ref
```
Also add `String` to the import of `Json.Decode` if not already present (it is — `string` is imported). We need `String.startsWith` and `String.dropLeft` which are from elm/core's `String` module (already available).

**2. Extend `definitionsDecoder`** per D-01 and D-02:
Replace the current `definitionsDecoder` with:
```elm
definitionsDecoder : Decoder Schema.Definitions
definitionsDecoder =
    let
        readDefs key =
            field key
                (keyValuePairs schemaDecoder
                    |> map (List.map (Tuple.mapFirst (\k -> "#/definitions/" ++ k)) >> Dict.fromList)
                )
                |> maybe
                |> map (Maybe.withDefault Dict.empty)
    in
    map2 Dict.union (readDefs "definitions") (readDefs "$defs")
```
Note: `Dict.union` is left-biased, so `definitions` keys win over `$defs` keys on conflict (D-02).

**3. Add `combinatorDecoder` helper** (new top-level function):
```elm
combinatorDecoder : Decoder (Maybe ( Schema.CombinatorKind, List Schema ))
combinatorDecoder =
    oneOf
        [ field "oneOf" (list schemaDecoder) |> map (\s -> Just ( Schema.OneOfKind, s ))
        , field "anyOf" (list schemaDecoder) |> map (\s -> Just ( Schema.AnyOfKind, s ))
        , field "allOf" (list schemaDecoder) |> map (\s -> Just ( Schema.AllOfKind, s ))
        , succeed Nothing
        ]
```
This references `schemaDecoder` which is already lazy, so mutual recursion is handled.
