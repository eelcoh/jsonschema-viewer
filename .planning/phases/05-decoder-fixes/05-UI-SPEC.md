## Scope Note

Phase 5 is a decoder fix phase. UI surface changes are limited to:

1. Combined type+combinator schemas (D-06, D-07): An existing parent pill node gains a second group of combinator children rendered below its regular property children. The combinator group uses the existing `viewMulti` pattern (combinator pill + branches). No new visual primitives are introduced.
2. `$defs` schemas now resolve definitions correctly — previously invisible references become visible as expanded subtrees. No new visual elements; the fix makes existing rendering work correctly for more schemas.

All color, typography, and spacing tokens are carried forward from the established v1.0 design without modification. This phase introduces no new visual design decisions.

## Interaction Contract

### Combined Schema Rendering (D-06, D-07)

When a schema has `combinator = Just (kind, subSchemas)`:

1. Parent pill renders at `(x, y)` as normal (object, array, string, etc.)
2. Regular children (properties/items) render to the right of the parent, starting at `(parentRightEdge + 10, y)`
3. After the last child, the combinator pill renders at `(parentRightEdge + 10, lastChildBottomY + ySpace)` where `ySpace` is the pre-existing inherited constant (10px, documented above)
4. Combinator sub-schemas render to the right of the combinator pill using the same `viewItems` pattern
5. A bezier connector path links the parent pill to the combinator pill (same `connectorPath` used for all other children)
6. The parent node's final dimensions use `Basics.max` across all child group widths and heights

### $defs Resolution (DEC-01)

No interaction change. Previously broken `$ref` links that returned `Nothing` now resolve to the target schema and render inline. From the user's perspective: schemas that previously showed only a `$ref` label now expand into full subtrees. No new interaction state is introduced.

### Collapse/Expand

Unaffected. The combinator group participates in the existing collapse/expand mechanism the same way `viewMulti` does for standalone combinator schemas today.
