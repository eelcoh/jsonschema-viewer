Per D-06 and D-07, extend `viewSchema` in `src/Render/Svg.elm` to check the `combinator` field on typed schemas and render the combinator group below the normal children.

**A. Add a helper function** `viewCombinatorGroup` that renders the combinator portion of a combined schema. Place it near `viewMulti`:

```elm
viewCombinatorGroup :
    Set String
    -> Definitions
    -> Set String
    -> (String -> msg)
    -> String
    -> Float
    -> Float
    -> Coordinates
    -> Schema.CombinatorKind
    -> List Schema
    -> ( List (Svg msg), Coordinates )
viewCombinatorGroup visited defs collapsedNodes toggleMsg path parentRightX parentY ( x, y ) kind subSchemas =
    let
        icon =
            case kind of
                Schema.OneOfKind -> "|1|"
                Schema.AnyOfKind -> "|o|"
                Schema.AllOfKind -> "(&)"

        combPath =
            path ++ ".combinator"

        ( combSvg, ( combW, combH ) ) =
            viewMulti visited defs collapsedNodes toggleMsg combPath ( x, y ) icon Nothing subSchemas

        connector =
            connectorPath ( parentRightX, parentY + 14 ) ( x, y + 14 )
    in
    ( [ connector, combSvg ], ( combW, combH ) )
```

**B. Update the `Schema.Object` arm** in `viewSchema` to check for `combinator` field. The current arm destructures `{ title, properties }`. Change to destructure `{ title, properties, combinator }`. In the expanded (non-collapsed) branch, after computing `(propertiesGraphs, (pw, ph))`, add:

```elm
case combinator of
    Nothing ->
        -- No combinator, render as before
        ( objectGraph :: propertiesGraphs, ( pw, Basics.max h ph ) )
            |> toSvgCoordsTuple

    Just ( kind, subSchemas ) ->
        let
            ( combGraphs, ( combW, combH ) ) =
                viewCombinatorGroup visited defs collapsedNodes toggleMsg path w y ( w + 10, ph + ySpace ) kind subSchemas
        in
        ( objectGraph :: propertiesGraphs ++ combGraphs
        , ( Basics.max pw combW, Basics.max h (Basics.max ph combH) )
        )
            |> toSvgCoordsTuple
```

The key coordinate threading: combinator group starts at `( w + 10, ph + ySpace )` where `w` is the object pill's right edge, `ph` is the bottom of the last property, and `ySpace = 10` is the standard vertical gap. The connector goes from `( w, y + 14 )` (parent pill's right midpoint) to `( w + 10, ph + ySpace + 14 )` (combinator pill's left midpoint).
