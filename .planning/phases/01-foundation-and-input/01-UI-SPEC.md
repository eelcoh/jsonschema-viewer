## Interaction Contract

### Textarea (INPUT-01, D-03, D-09)

| State | Visual |
|-------|--------|
| Default | Border `#d0d7de`, background `#ffffff` |
| Focused | Border `#0969da` (2px), background `#ffffff` |
| Valid input | Border `#d0d7de` (no explicit success state — diagram updating is the feedback) |
| Invalid input (errors shown) | Border `#cf222e` (2px); error text replaces diagram |
| Invalid input (debounce pending) | Border `#d0d7de`; last valid diagram remains visible |

- Textarea occupies 100% of the left panel height minus toolbar
- `resize: none` — panel width controls sizing, not textarea drag handle
- `font-size: 13px`, `line-height: 1.6`, `padding: 16px`
- `spellcheck: false`, `autocorrect: off`, `autocapitalize: off`

### Drag-and-Drop Zone (INPUT-02, D-04)

| State | Visual |
|-------|--------|
| Default | No visual distinction — the textarea IS the drop zone |
| Dragover (file over textarea) | Textarea background `#dbeafe`, border `#0969da` (2px) |
| Drop accepted | Background returns to `#ffffff`; textarea content replaced with file content |
| Drop rejected (non-.json) | Background `#fff3cd`, border `#d97706` for 300ms then returns to default; no error message for Phase 1 (file type filtering deferred) |

- Accepted file types for Phase 1: any `.json` file (MIME type `application/json` or filename ending `.json`)
- Multiple files dropped: first file is used; extras are ignored silently

### Panel Collapse/Expand (D-02)

| State | Visual |
|-------|--------|
| Expanded (default) | Left panel visible; toggle shows `<` arrow or "Hide input" label |
| Collapsed | Left panel `display: none`; toggle shows `>` arrow or "Show input" label; toggle button stays visible at left edge of diagram area |

- Toggle element: a `<button>` in the toolbar, 32px × 32px, no background, accent text color (`#0969da`)
- Toggle label text: `Hide` when panel is open, `Show` when panel is closed
- Transition: no animation in Phase 1 (snap, not slide)

### Example Selector (D-10, D-11)

- Rendered as a button group in the toolbar, not a `<select>` element
- Three buttons: `Arrays`, `Person`, `Nested`
- Button appearance:
  - Default: background `#f6f8fa`, border `#d0d7de`, text `#293c4b`, 28px height, 8px horizontal padding
  - Active (selected): background `#0969da`, border `#0969da`, text `#ffffff`
- Clicking a button replaces textarea content and triggers immediate decode + render
- One example is always active; initial active example is `Arrays` (maps to existing `jsonschema` string — D-07)

### Debounce Behavior (D-09)

- Debounce delay: 800ms (from RESEARCH.md Pattern 2)
- On every keystroke: diagram shows last valid schema; no error shown
- After 800ms of no input: if current parse is `Err`, error replaces diagram area
- When user resumes typing: error clears immediately; last valid diagram returns

## States and Error Display

### Diagram Area States

| State | What Renders |
|-------|-------------|
| Valid schema | SVG diagram via `Render.Svg.view` |
| Debounce pending (invalid) | Last valid SVG diagram (D-09) |
| Error displayed | Error message block (see below) |
| Initial load | `Arrays` example schema SVG diagram (D-07) |

### Error Display (D-08)

Error replaces the entire diagram area. Layout:

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  Invalid JSON Schema                            │  ← 20px, weight 600, #cf222e
│                                                 │
│  The pasted text is not valid JSON Schema.      │  ← 14px, weight 400, #293c4b
│  Check the input and try again.                 │
│                                                 │
│  [error detail from Json.Decode.errorToString]  │  ← 13px, weight 400, #293c4b,
│                                                 │     font-family: monospace,
│                                                 │     background: #f6f8fa,
│                                                 │     padding: 16px, overflow-x: auto
└─────────────────────────────────────────────────┘
```

Error container: centered vertically and horizontally in diagram area, max-width 560px, padding 32px.

## Component Inventory

All components are plain Elm `Html` — no component library. Each maps to an Elm view function.

| Component | Elm View Function | Description |
|-----------|-------------------|-------------|
| Toolbar | `viewToolbar` | 48px fixed-height bar with app name, example buttons, collapse toggle |
| Left panel | `viewInputPanel` | Textarea + drag-drop zone; collapses on toggle |
| Textarea | rendered inside `viewInputPanel` | Full-height textarea for schema input |
| Drag overlay | rendered inside `viewInputPanel` | `onFileDrop` + `onDragOver` attributes on textarea container div |
| Example button group | `viewExampleButtons` | Three buttons inside toolbar |
| Right panel | `viewDiagramPanel` | SVG diagram or error state |
| Error display | `viewError` | Error heading + body + decoded error detail |
| Collapse toggle | `viewCollapseToggle` | Single button in toolbar |

## Elm-Specific Rendering Notes

- Elm SVG is rendered via `Render.Svg.view : Definitions -> Schema -> Html msg` — this function is unchanged in Phase 1
- The diagram area must be a `div` with `overflow: auto` to allow scroll if SVG exceeds viewport (SVG viewport fix is Phase 2)
- The textarea `onInput` handler delivers the full string on every keystroke — no incremental parsing
- `Html.Attributes.attribute "spellcheck" "false"` is required to suppress browser spell-check on JSON
- The `placeholder` attribute on textarea is the only "empty state" for the input panel
